# RUNBOOK

## 项目简介
DouyinLiveRecorder 是一个可长期值守的抖音直播录制工具，现已增强为带内置 Web 面板的值守面板版本。

## 运行方式

### 1. Docker
```bash
docker compose up -d
```
- 默认映射端口：`18080:18080`
- 配置目录：`./config`
- 下载目录：`./downloads`

### 2. CLI / Python
```bash
python3 main.py
```
如使用虚拟环境：
```bash
. .venv/bin/activate
python3 main.py
```

### 3. EXE
1. 准备打包版发布包（通常来自 Releases）。
2. 编辑 `config/config.ini` 和 `config/URL_config.ini`。
3. 双击运行 `DouyinLiveRecorder.exe`。

## Web 面板访问地址
- 默认端口：`18080`
- 当前项目配置为监听 `0.0.0.0:18080`
- 本机访问：`http://127.0.0.1:18080`
- 局域网 / Docker 宿主机访问：`http://<主机IP>:18080`

## 面板重点能力
- 总览页：抖音直播状态、最近录制、异常提示、磁盘告警
- 录制文件页：按房间名 / 日期 / 关键词筛选，支持大小/时间排序
- `config.ini` 在线编辑
- `URL_config.ini` 在线编辑与预览

## 常见问题排查

### 1. Web 面板打不开
- 确认 `config/config.ini` 中 `是否启用Web控制台(是/否) = 是`
- 确认端口 `18080` 未被占用
- Docker 场景确认已映射 `18080:18080`
- 若改过监听地址/端口，重启主程序后再访问

### 2. 直播间一直不录制
- 检查 `config/URL_config.ini` 是否为空
- 检查目标行前是否带 `#` 注释
- 检查抖音 Cookie 是否已配置且未失效
- 检查代理设置是否符合目标平台需求

### 3. 配置改了没生效
- `URL_config.ini` 通常约 3 秒内热更新
- 多数录制参数会在下一轮轮询或下一次新录制时生效
- Web 控制台开关、监听地址、监听端口等启动级配置需重启程序

### 4. 面板显示磁盘告警
- 使用率 `>80%` 会出现黄色告警
- 使用率 `>90%` 会出现红色横幅
- 请清理 `downloads` 旧文件，或切换到更大磁盘

### 5. 录制文件页看不到文件
- 先确认 `downloads` 目录下确实已有录制产物
- 若下载目录被改到自定义路径，检查 `直播保存路径(不填则默认)`
- 若文件很多，可提高 `Web控制台文件索引上限`

### 6. 异常提示频繁出现
- 优先查看总览页“异常提示”和“关键事件”
- 常见原因：Cookie 失效、直播源请求失败、ffmpeg 异常退出、磁盘不足
- 结合 `logs/streamget.log` 与 `logs/PlayURL.log` 进一步定位
